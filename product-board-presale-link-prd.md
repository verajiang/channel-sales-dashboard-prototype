# 商品看板预售与链接维度优化需求文档

## 1. 背景

当前万象商品看板已有 `SPU维度` 和 `SKU维度`，用于查看商品销售、实付、退款、库存等表现。业务方在商品经营分析中还需要同时判断两类问题：

1. 商品销售中预售与现货分别贡献多少销售额、实付金额和件数。
2. 同一商品通过不同商品链接产生销售时，需要按链接查看表现，并继续拆分该链接下自营与达人贡献。

因此本次需求在现有商品看板基础上补充预售/现货指标，并新增 `链接维度`。

线上原型地址：
https://verajiang.github.io/channel-sales-dashboard-prototype/product-board-presale-link-prototype.html

## 2. 目标

1. 在商品销售明细中补充预售数据、现货数据和现货预售占比，支持业务判断商品销售结构。
2. 在 `SPU维度`、`SKU维度` 外新增 `链接维度`，支持按商品链接聚合查看销售表现。
3. 链接维度下，一行代表一个商品链接；点击链接行可展开该链接下 `自营` 和 `达人` 两类数据。
4. 链接维度表格字段与展开行保持一致，避免主行与明细行口径不一致。
5. 避免重复展示信息：商品链接与关联商品合并为一列，不再单独展示平台列和 SKU 操作入口。

## 3. 用户与场景

主要用户：
- 商品运营：判断商品在不同链接下的销售、预售、退款和库存表现。
- 渠道运营：对比自营与达人对同一链接的销售贡献。
- 店铺负责人：查看商品链接层级的销售结构，定位预售占比、退款率或库存异常。
- 数据/研发团队：根据明确字段和交互规则进行接口与页面开发。

核心场景：
1. 业务方按月份、店铺、抖音ID、货号筛选商品表现。
2. 商品运营从 `SPU维度` 或 `SKU维度` 切换到 `链接维度`，查看每个商品链接的销售明细。
3. 业务方点击某个商品链接，展开查看该链接下自营与达人的销售、预售、退款、库存拆分。
4. 业务方导出当前筛选条件下的链接维度数据，用于复盘或二次分析。

## 4. 页面范围

### 4.1 维度切换

商品看板顶部维度切换包含：
- `SPU维度`
- `SKU维度`
- `链接维度`

默认维度保持现有逻辑不变；进入 `链接维度` 后，标题展示为：

```text
商品销售明细（链接维度）
```

### 4.2 筛选区

链接维度复用商品看板现有筛选能力：
- 时间快捷筛选：最近30日、年、月、周、昨天、今天、自定义
- 日期选择
- 店铺筛选
- 抖音ID筛选
- 货号筛选
- 搜索
- 导出

说明：
- 筛选条件作用于当前维度表格。
- 货号筛选仍按商家编码/货号口径过滤商品范围，不作为链接下钻入口。

## 5. 链接维度表格

### 5.1 表格粒度

链接维度主表粒度为 `商品链接`。

规则：
- 一行代表一个商品链接。
- `全部商品链接` 为合计行，汇总当前筛选条件下所有链接数据。
- 每个商品链接可展开两类明细：`自营`、`达人`。
- 展开明细行与主链接行使用同一套指标字段。

### 5.2 首列信息

首列名称：

```text
商品链接及关联商品
```

首列展示内容：
- 展开/收起按钮
- 平台来源标识，如抖音、淘宝、视频号，以图标或短标识展示
- 商品链接名称，可点击打开原始商品链接
- 关联商品名称
- 商品ID

说明：
- 商品链接与关联商品合并展示，避免拆成两列导致信息重复。
- 平台只作为首列内的来源标识，不再单独设置 `平台` 列。
- 下方小字展示 `商品ID`，不展示 SKU 数或商家编码聚合信息。
- 链接维度不提供 `查看SKU` 操作，因为 SKU 页按商家编码聚合，与链接维度下钻口径不一致。

### 5.3 字段分组

链接维度表格字段如下。

固定列：
- 商品链接及关联商品

销售合计：
- 销售金额
- 销售件数
- 实付金额
- 实付件数

预售数据：
- 预售GMV
- 预售实付
- 预售件数

现货数据：
- 现货GMV
- 现货实付
- 现货件数

占比：
- 现货预售占比

退款数据：
- 总退款率
- 售前退款率
- 售后退款率
- 现货退款率
- 预售退款率

库存数据：
- 现货库存
- 采购在途

## 6. 交互规则

### 6.1 展开与收起

触发方式：
- 点击链接行左侧展开按钮。

展开后：
- 在当前链接行下方展示 `自营` 和 `达人` 两行。
- 明细行首列展示 `自营` 或 `达人` 标签。
- 明细行字段与主链接行字段保持一致。

收起后：
- 隐藏该链接下的 `自营` 和 `达人` 明细行。

### 6.2 横向滚动与固定列

表格字段较多，需支持表格内部横向滚动。

要求：
- 页面整体不出现横向滚动。
- 横向滚动条只出现在表格区域内。
- `商品链接及关联商品` 作为首列固定，横向滚动时保持可见。
- 表头在表格内部纵向滚动时保持可见。

### 6.3 排序

当前原型未明确链接维度排序控件。本期建议至少保留现有商品看板默认排序逻辑。

待确认：
- 链接维度是否需要支持按销售金额、销售件数、实付金额、退款率等字段排序。
- 如果支持排序，合计行需始终固定在首行，不参与排序。

## 7. 指标口径

以下口径基于当前原型字段与既有看板 PRD 中退款率口径整理；具体数据源字段映射需在开发前由数据侧确认。

### 7.1 销售金额

```text
销售金额 = 预售GMV + 现货GMV
```

说明：
- 用于衡量该链接在当前筛选条件下的销售规模。
- 当前原型中预售GMV与现货GMV相加等于销售金额。

### 7.2 销售件数

```text
销售件数 = 预售件数 + 现货件数
```

说明：
- 用于衡量该链接在当前筛选条件下的销售件数。

### 7.3 实付金额

```text
实付金额 = 预售实付 + 现货实付
```

说明：
- 用于衡量该链接在当前筛选条件下的实付表现。
- 与综合看板中的 GMV 口径是否完全一致，需要结合商品看板现有口径确认。

### 7.4 实付件数

```text
实付件数 = 预售实付件数 + 现货实付件数
```

说明：
- 当前页面字段名为 `实付件数`，预售/现货拆分字段分别展示为 `预售件数`、`现货件数`。
- 如果数据侧区分“销售件数”和“实付件数”，需在接口字段中明确返回两个口径。

### 7.5 现货预售占比

```text
现货占比 = 现货GMV / 销售金额
预售占比 = 预售GMV / 销售金额
```

展示格式：

```text
现货 xx.x% / 预售 xx.x%
```

说明：
- 当销售金额为 0 时，展示为 `-`。

### 7.6 总退款率

```text
总退款率 = 总退款金额 / 销售金额
```

说明：
- 总退款金额包含售前退款金额和售后退款金额。
- 当销售金额为 0 时，展示为 `-`。

### 7.7 售前退款率

```text
售前退款率 = 售前退款金额 / 销售金额
```

说明：
- 用于判断未发货前产生的退款压力。
- 既有全渠道 PRD 中同类字段名称为 `发货前退款率`，商品看板页面使用业务侧截图中的 `售前退款率` 文案。

### 7.8 售后退款率

```text
售后退款率 = 售后退款金额 / 销售金额
```

说明：
- 用于判断发货后由履约、商品体验等因素产生的退款压力。
- 既有全渠道 PRD 中同类字段名称为 `发货后退款率`，商品看板页面使用业务侧截图中的 `售后退款率` 文案。

### 7.9 现货退款率

```text
现货退款率 = 现货退款金额 / 现货GMV
```

说明：
- 当现货GMV为 0 时，展示为 `-`。
- 现货退款金额的数据源字段需由数据侧确认。

### 7.10 预售退款率

```text
预售退款率 = 预售退款金额 / 预售GMV
```

说明：
- 当预售GMV为 0 时，展示为 `-`。
- 预售退款金额的数据源字段需由数据侧确认。

### 7.11 现货库存

```text
现货库存 = 当前可售现货库存数量
```

说明：
- 展示当前筛选条件下该链接关联商品的现货库存。
- 如果一个商品链接关联多个商品或多个 SKU，库存汇总规则需由数据侧确认。

### 7.12 采购在途

```text
采购在途 = 已采购未入库数量
```

说明：
- 用于判断未来可补充库存。
- 与现货库存的汇总粒度需保持一致。

## 8. 导出需求

链接维度支持导出当前筛选条件下的数据。

建议导出字段：
- 日期范围
- 店铺
- 商品链接名称
- 商品链接URL
- 平台来源
- 商品ID
- 关联商品名称
- 数据类型：合计 / 自营 / 达人
- 销售金额
- 销售件数
- 实付金额
- 实付件数
- 预售GMV
- 预售实付
- 预售件数
- 现货GMV
- 现货实付
- 现货件数
- 现货占比
- 预售占比
- 总退款率
- 售前退款率
- 售后退款率
- 现货退款率
- 预售退款率
- 现货库存
- 采购在途

说明：
- 页面不单独展示平台列，但导出建议保留 `平台来源`，方便二次分析。
- 展开明细中的 `自营` 和 `达人` 建议通过 `数据类型` 字段表达。

## 9. 非功能要求

1. 表格需保持后台看板风格，优先保证可读性和信息密度。
2. 字段较多时使用表格内部横向滚动，避免整个页面横向滚动。
3. 首列固定，确保业务方滚动查看右侧指标时仍能识别当前链接。
4. 不展示面向研发或原型说明性质的提示文案。
5. 不重复展示平台信息，不新增与链接维度口径不一致的 SKU 下钻入口。
6. 金额、件数、比例字段需右对齐并使用千分位展示。
7. 比例字段保留 1 到 2 位小数，具体精度与现有商品看板保持一致。

## 10. 验收标准

1. 商品看板支持 `SPU维度`、`SKU维度`、`链接维度` 三个维度切换。
2. 进入 `链接维度` 后，表格标题展示为 `商品销售明细（链接维度）`。
3. 链接维度主表一行代表一个商品链接。
4. `商品链接及关联商品` 合并展示链接名称、关联商品名称、商品ID和平台来源标识。
5. 页面不单独展示 `平台` 列。
6. 页面不展示 `查看SKU` 操作。
7. 点击商品链接行可展开 `自营` 和 `达人` 两行数据。
8. 展开行与主链接行字段一致。
9. 表格展示销售合计、预售数据、现货数据、现货预售占比、退款数据和库存数据。
10. 退款数据包含总退款率、售前退款率、售后退款率、现货退款率、预售退款率。
11. 库存数据包含现货库存和采购在途。
12. 页面整体不出现横向滚动，横向滚动只发生在表格区域内。
13. 横向滚动时首列保持固定。
14. 表头在表格区域纵向滚动时保持可见。
15. 导出数据包含链接合计、自营、达人三类数据，并与页面字段口径一致。

## 11. 后续待确认

1. 链接维度默认排序字段。
2. 链接维度是否需要支持点击表头排序。
3. 商品链接与商品ID、SKU、货号之间的数据映射关系。
4. 自营/达人在不同平台下的识别规则。
5. 现货GMV、预售GMV、现货实付、预售实付的源表字段。
6. 现货退款金额、预售退款金额的源表字段。
7. 一个链接关联多个商品或多个 SKU 时，现货库存与采购在途的汇总规则。
8. 导出是否默认包含收起状态下不可见的自营/达人明细行。
9. 比例字段小数位是否统一保留 2 位。
