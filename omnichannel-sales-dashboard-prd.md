# 全渠道销售综合看板产品需求文档

## 1. 背景

当前业务方已有一个全渠道销售综合看板，用于查看不同平台店铺、商品的销售表现。随着业务分析需求增加，原看板仅展示销售额、订单数、件数、退款等基础指标，无法充分解释不同渠道、经营类型和销售场景的表现差异。

本次需求基于当前原型沉淀，目标是补充关键经营字段、统一指标口径，并支持排行数据导出，方便业务方在线分享、评审和后续研发落地。

线上原型地址：
https://verajiang.github.io/channel-sales-dashboard-prototype/

## 2. 目标

1. 统一 GMV 口径为用户实际支付金额。
2. 在概览区突出核心经营指标：GMV、净销售额、推广金额、支付订单数。
3. 增加渠道经营拆分，支持按平台、经营类型、销售场景查看经营表现。
4. 补充发货前退款率、发货后退款率、预售金额占比，提升退款结构和预售结构判断能力。
5. 支持店铺排行、商品排行、渠道经营拆分的数据导出。

## 3. 用户与场景

主要用户：
- 业务运营：查看各平台销售表现、退款表现、推广投入情况。
- 渠道负责人：对比抖音、淘宝、天猫、视频号等渠道表现。
- 商品运营：查看商品销售排行、退款率和件单表现。
- 数据/研发团队：根据明确字段和口径进行数据接口开发。

核心场景：
1. 业务方按日期、店铺、平台筛选销售表现。
2. 运营查看全渠道经营概览，快速判断 GMV、净销售额、推广费比是否异常。
3. 渠道负责人查看抖音自营/达人、直播间/商品卡，以及其他渠道自营/达人表现。
4. 业务方导出排行或拆分数据，用于日报、复盘或二次分析。

## 4. 指标口径

### 4.1 GMV

本需求中的 GMV 定义为用户实际支付金额。

```text
GMV = 用户实际支付金额
```

说明：
- GMV 与此前“实付金额”同口径。
- 页面不再单独展示“实付金额”字段。

### 4.2 净销售额

```text
净销售额 = GMV - 退款金额
```

说明：
- 用于衡量扣除退款后的实际销售表现。
- 概览区展示净销售额及环比。

### 4.3 推广费比

```text
推广费比 = 推广金额 / 净销售额
```

说明：
- 概览、店铺排行、渠道经营拆分均按此口径计算费比。
- 当净销售额为 0 时，费比展示为 `-`。

### 4.4 退款率

```text
退款率 = 退款金额 / GMV
```

说明：
- 当前渠道经营拆分中的退款率不做颜色预警。
- 后续待项目组确认预警规则后，再增加标红、橙色等状态。

### 4.5 发货前退款率

```text
发货前退款率 = 发货前退款金额 / GMV
```

说明：
- 用于区分未发货前产生的退款压力。
- 经营概览和店铺销售排行展示该比例。

### 4.6 发货后退款率

```text
发货后退款率 = 发货后退款金额 / GMV
```

说明：
- 用于区分发货后由履约、商品体验等因素产生的退款压力。
- 经营概览和店铺销售排行展示该比例。

### 4.7 预售金额占比

```text
预售金额占比 = 预售GMV / GMV
```

说明：
- 用于衡量当前 GMV 中预售订单贡献。
- 当前仅在经营概览展示。

## 5. 页面范围

### 5.1 筛选区

支持筛选项：
- 时间快捷筛选：最近30日、年、月、周、昨天、今天、自定义
- 日期选择
- 店铺筛选
- 平台筛选

### 5.2 销售总览

主指标：
- GMV
- GMV 环比
- GMV 口径说明：用户实际支付金额

并排卡片：
- 净销售额
- 净销售额环比
- 推广金额
- 推广费比
- 支付订单数
- 客单价

底部辅助指标：
- 支付件数
- 件单价
- 退款金额
- 退款率
- 发货前退款率
- 发货后退款率
- 预售金额占比
- 订单连带率
- 主力店贡献

不展示：
- 实付金额独立字段

### 5.3 渠道经营拆分

字段：
- 平台
- 经营类型
- 场景
- GMV
- 净销售额
- 退款率
- 推广金额
- 推广费比

拆分规则：
- 抖音：
  - 抖音合计
  - 自营合计
  - 自营 / 直播间
  - 自营 / 商品卡
  - 达人合计
  - 达人 / 直播间
  - 达人 / 商品卡
- 天猫：
  - 天猫合计
  - 订单商品标题包含 `【】` 时，识别为达播订单
  - 订单商品标题不包含 `【】` 时，识别为自营订单
  - 按识别结果汇总自营 / 整体、达人 / 整体
- 其他渠道：
  - 渠道合计
  - 自营 / 整体
  - 达人 / 整体

当前包含渠道：
- 抖音
- 淘宝
- 天猫
- 视频号

交互：
- 支持导出。
- 右上角不提供“按经营类型”等切换按钮。

### 5.4 店铺销售排行

字段：
- 排名
- 平台店铺
- GMV
- 净销售额
- 订单 / 客单
- 件数 / 件单
- 退款
- 发货前退款率
- 发货后退款率
- 推广金额 / 推广费比
- 销售占比

排序入口：
- 按 GMV
- 按净销售额
- 按推广金额

不展示：
- 实付金额

交互：
- 支持导出。

### 5.5 商品销售排行

字段：
- 排名
- 商品
- 平台
- GMV
- 净销售额
- 件数 / 件单
- 退款

排序入口：
- 按 GMV
- 按件数
- 按退款率

不展示：
- 实付金额
- 品类
- 系列
- 推广金额
- SKU ID
- 商品规格

交互：
- 支持导出。

## 7. 非功能要求

1. 页面需支持在线分享，当前通过 GitHub Pages 发布。
2. 页面应保持后台看板风格，避免过度装饰。
3. 表格字段需保证可读性，避免重复展示同口径指标。
4. 导出字段必须与页面展示口径一致。
5. 退款率暂不做预警色，待业务规则明确后再扩展。

## 8. 验收标准

1. 概览区 GMV 展示为用户实际支付金额口径。
2. 页面中不再出现“实付金额”字段。
3. 推广费比按 `推广金额 / 净销售额` 计算。
4. 渠道经营拆分包含抖音自营/达人及直播间/商品卡拆分。
5. 淘宝、天猫、视频号均包含渠道合计、自营、达人行。
6. 天猫自营/达人按订单商品标题是否包含 `【】` 识别：包含为达播订单，不包含为自营订单。
7. 渠道经营拆分退款率不标红。
8. 经营概览展示发货前退款率、发货后退款率、预售金额占比。
9. 店铺销售排行展示发货前退款率、发货后退款率。
10. 店铺排行、商品排行、渠道经营拆分均可导出。
11. 三份导出模板字段与页面口径一致。
12. GitHub Pages 链接可访问并展示最新版本。

## 9. 后续待确认

1. 各渠道退款率预警规则。
2. 抖音直播间、商品卡、自营、达人数据源字段映射。
3. 其他渠道是否未来也需要拆分到更细销售场景。
4. 推广金额是否包含平台佣金、达人佣金或仅包含投放消耗。
5. 销售占比的分母是否始终为全渠道 GMV。
6. 发货前退款金额、发货后退款金额、预售GMV在各平台的数据源字段映射。
